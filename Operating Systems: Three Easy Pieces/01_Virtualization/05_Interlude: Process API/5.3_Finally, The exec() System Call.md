프로세스 생성 API의 마지막이자 중요한 부분은 exec() system call이다.

이 system call은 호출 프로그램과 다른 프로그램을 실행하고 싶을 때 유용하다.

예를 들어, fork() 호출은 같은 프로그램의 복사복을 계속 실행하고 싶을 때만 유용하다.

**그러나 종종 다른 프로그램을 실행하고 싶을 수 있는데, 이때 exec()를 쓰면 된다.**

<img width="500" alt="" src="https://github.com/user-attachments/assets/653013bc-27d1-4891-a234-b78ce3394b17" />

위 예에서 자식 프로세스는 단어 계산 프로그램인 wc 프로그램을 실행하기 위해 execvp()를 호출한다.

<img width="500" alt="" src="https://github.com/user-attachments/assets/00d03221-57ef-43e0-8b9d-aef5e324258b" />

exec() system call은 실행 파일(예: wc)의 이름과 일부 인수(예: p3.c)가 주어지면, 해당 실행 파일에서 코드(및 정적 데이터)를 로드하고 현재 코드 세그먼트(및 현재 정적 데이터)를 덮어쓴다.

힙과 스택 및 프로그램의 메모리 공간의 다른 부분은 다시 초기화된다.

그런 다음 OS는 단순히 그 프로그램을 실행하고, 모든 인수를 해당 프로세스의 argv로 전달한다.

따라서 그것은 새로운 프로세스를 만들지 않는다.

오히려 그것은 현재 실행 중인 프로그램(이전의 p3)을 다른 실행 중인 프로그램(wc)으로 변환한다.

자식의 exec() 이후, 마치 p3.c가 실행되지 않은 것과 같다.
